# 委任仕様テンプレート

サブエージェントやヘルパーAIにタスクを委任するたびに、このテンプレートを使用してください。

---

## 12の必須項目

委任前に12項目すべてを記入してください。項目の欠落は、スコープクリープ、権限違反、品質低下につながります。

| # | 項目 | 値 |
|---|------|---|
| 1 | **タスク名** | |
| 2 | **委任レベル** | D1（アシスタントAI） / D2（MCP） / D3（フルパイプライン） |
| 3 | **スコープ** | （正確に何をするか -- それ以上でも以下でもなく） |
| 4 | **スコープ除外** | （明示的に何をしないか） |
| 5 | **前提条件** | （開始前に何が真でなければならないか） |
| 6 | **権限** | （Read/Write/Execute -- 対象ファイル/API/コマンド） |
| 7 | **権限拒否** | （明示的に何が許可されていないか） |
| 8 | **期待出力** | （フォーマット、内容、出力先） |
| 9 | **品質基準** | （出力が合格かどうかをどう判断するか） |
| 10 | **タイムアウト** | （エスカレーションまでの最大時間） |
| 11 | **ロールバック条件** | （いつ中止し、どう元に戻すか） |
| 12 | **報告フォーマット** | （何を、どの構造で報告するか） |

## 委任レベルガイド

| レベル | 使うタイミング | 例 |
|--------|--------------|---|
| **D1** | 定型監視、単純なクエリ | ログスキャン、ヘルスチェック |
| **D2** | 定義済みインターフェースを持つ構造化ツール利用 | MCPベースのファイル操作 |
| **D3** | エンドツーエンドのタスク実行 | API Gateway経由のフルパイプライン |

## 委任後の検証

- [ ] 出力が期待出力仕様と一致しているか
- [ ] 権限違反が発生していないか
- [ ] スコープが仕様を超えて拡大されていないか
- [ ] 品質基準を満たしているか
- [ ] 所要時間がタイムアウト内であるか

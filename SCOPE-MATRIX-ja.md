## 公開版：提供するもの・しないもの

### 公開版が提供するもの（5点）

1. 即座のハンズオン体験（Control OS -- 30秒で試せる）
2. 完全な構造的方法論（10-framework/ -- なぜ機能するか）
3. 公開検証面（PROVE-IT -- 検査して異議を申し立て可能）
4. 移植キット（テンプレート -- 自分の環境で構築）
5. 明確なスコープ境界（本文書 -- 何を得られるか把握）

### 公開版が提供しないもの（5点）

1. 完全な実装詳細の提供（Phase1/Phase2の範囲）
2. すべての定量的証明（一部のN数は公開範囲外）
3. ハンズオン体験の代替（自分で試す必要あり）
4. 個人の経歴情報（方法論のプロジェクトであり、伝記ではない）
5. 結果の保証（自分の環境で検証せよ）

## Phase1: 非公開の参照資料（骨格版）

**焦点**: 方法論、ステップごとの思考法、判断基準、ケーススタディ。

**役割**: 方法論の教授、判断基準の提供、設計根拠の提示、適応の支援。

含む: 設計文書10点 + FM 60項目 + 5層/3層図 + サンプル会話 + reason_codeテーブル + 注釈付き代表画像 + N数・検知率。

含まない: 全132 FM項目、完全ソースコード、完全運用ログ、再現キー。

## Phase2: 非公開の参照資料（完全版）

**焦点**: より深い構造観察手法、完全な分析、完全な証拠。

**役割**: 完全な構造観察、保守方法論、深い分析、完全な証拠基盤。

含む: 全132 FM項目 + 全設計ファイル + 全画像 + 完全品質システム（600行超） + 検証ログ + ソース抜粋 + 構造観察方法論 + 完全世代ログ。

含まない: 再現キー（パス、APIキー、認証情報）、個人情報、本番デプロイ用完全ソースコード。

Phase1 / Phase2 の記載は、公開範囲の境界を明確にするためのものです。評価の主対象は公開版リポジトリです。

## 書籍: 体系的論考

**焦点**: AI統治のためのSHI理論。7章による体系的論考。

**役割**: ナラティブの一貫性、理論的フレームワーク、再現手順、概念図。

書籍は「設計文書の束を、進化の読める記録に変える媒体」である。7章の構成: 問題発見、完全記憶、外部監視、5層運用、品質システム、証拠収集、SHI理論の実践。読者が類似システムを構築するための再現手順（特定のキーなし）を含む。

含まない: 実際のコードやスクリプト、再現キー、生の運用データ、個人情報。独立した著作として設計。

## GitHub公開ルール（5点）

1. すべての公開主張はエビデンスにリンクするか、公開エビデンスの境界を明示的にラベル付けしています
2. 公開ファイルに内部パス、エンドポイント、個人情報を含めない
3. スター評価は著者の自己評価ではなく、公開検証可能性で定義する
4. 未検証の主張には「proven」ではなく「著者の環境で観察」を使用する
5. 公開範囲外の内容は「何を含むか」で記述し、内容そのものは抜粋しない

## 読者への期待

本リポジトリはあなたに以下を求めます：
1. **試す** -- Control OSを自分のAIで使ってみる
2. **検証する** -- 方法論が自分の環境で機能するか確認する
3. **異議を申し立てる** -- 根拠が不十分な主張に対して
4. **構築する** -- テンプレートと方法論を使って自分の統治システムを作る
5. **判断する** -- 公開リポジトリだけで、自分が試す・反証する・転用するのに十分かを判断する

# コントリビューションガイド

本リポジトリは**方法論リポジトリ**であり、ソフトウェア製品ではありません。貢献は歓迎します。特にIssueを通じた貢献が最も価値があります。

## 貢献方法

本プロジェクトは**Issues-firstワークフロー**を採用しています。実環境での観測は最も価値のある貢献です。多様なAIモデルや利用コンテキストに対して方法論を検証（または反証）する助けになるからです。

### 観測報告

Control OSやFM-40チートシートをご自身の環境で試して何かを観測した場合（効果あり・部分的・効果なし問わず）、以下の情報でIssueを作成してください:

- AIモデル（Claude、GPT、Copilot等）
- テスト内容（Control OS、特定FM観測等）
- 観測結果
- 環境コンテキスト（該当する場合）

### 主張への反証

本リポジトリのすべての主張は検証可能であることを意図しています。裏付けのない主張、誤解を招く記述、またはご自身の観測と矛盾する主張を発見した場合、Issueを作成してください。検証プロトコルは [PROVE-IT-ja.md](PROVE-IT-ja.md) を参照してください。

### 構造的改善の提案

方法論の構造（単なる文言ではなく）を改善する方法がある場合、以下を記述してください:

- 現在欠けているまたは壊れている構造的条件
- その障害を取り除く変更内容
- それが読者体験をどう改善するか

## Issueを出した後どうなるか

あなたのIssueは以下のフローで処理されます:

1. **トリアージ** — 著者がレポートを読み、ラベル付け（観測/反証/構造改善）
2. **再現・境界確認** — 著者が自環境で再現を試みるか、主張/境界の正確性を確認
3. **統合判断** — 確認された場合、以下のいずれかに反映:
   - **FM（Failure Mode）分類の改訂**（新FM追加または既存FM精緻化）
   - **PROVE-IT検証注記の更新**（主張マップの更新または検証ステータスの変更）
   - **SCOPE-MATRIX境界の明確化**（Free/Phase1/Phase2の範囲調整）
   - **テンプレート改善**（Control OSや導入テンプレートの改良）
4. **クローズ** — 更新されたドキュメントへのリンク付きでクローズ、または非統合の理由を記載

あなたの観測は、このリポジトリを使う全員のために方法論を直接改善します。

### Pull Requestとして受け付けるもの
- タイプミスの修正
- リンク切れの修正
- 翻訳品質の改善（en/jaミラーの整合性を保つこと）

方法論のコア内容は著者がキュレーションします。ドキュメント修正は歓迎します。

## 受け付けないもの

- **コンテンツ変更のPull Request**: 方法論リポジトリであり、特定の設計判断が関わります。コンテンツ変更はIssueで提案してください。著者が検証された改善を統合します
- **自動翻訳修正**: 翻訳品質はen/jaミラープロセスの一部として管理されています
- **ソフトウェア機能のリクエスト**: 本リポジトリには実行可能なソフトウェアは含まれていません

## 開発者向け注記

- **ファイル命名規則**: 英語専用ファイルは `-en` サフィックス付き（例: `control-os-claude-en.md`）。サフィックスなしのファイルは日本語版または両言語共用です。

## ライセンスとワークフロー

Fork / 再利用 / 派生著作物はMITライセンスの下で完全に許可されています。本リポジトリの正規バージョンではIssues-firstワークフローを採用しています: 観測を報告し、Issueで変更を提案し、著者が検証された改善を統合します。

## 行動規範

敬意を持って対応してください。意見ではなく観測に焦点を当ててください。主張ではなく証拠で反論してください。

# 失敗傾向の分類（Claude vs ChatGPT）
Language: [English version](../../10-framework/01-failure-modes.md)

> **観測期間**: 2025年後半〜2026年3月。すべてのモデル名はこの期間中に利用可能だったバージョンを指します。
> **適用範囲**: すべての行動評価（Low/Medium/High）は単一環境・単一運用者による観測です。ベンチマークではありません。検証可能な仮説として扱ってください。


Claude と ChatGPT の「観測できる失敗パターン」を、共通のFMインデックスとして定義します。

> **クイックスタート**: まずFM-01〜FM-10を読んでください。観測されたAIモデルで最も頻繁に観測されるパターンです。FM-11〜FM-40はより深い構造的カバレッジを提供します。

本ドキュメントの方針：
- **FM-01..FM-40** を固定IDとして扱う（言語・ツールが変わっても同一参照）。
- **日本語名が正本キー**（英語ラベルは `en/` 側）。
- **公開安全**：資格情報／エンドポイント／絶対パス／識別子は出さない。

> **注記：LLM構造的限界のマップとして** — 本FM体系は単なるミスの一覧ではなく、LLMの構造的限界を体系化した「配布可能なパッチセット」です。公開版のFM体系からより包括的な構造分解 [observed: single environment] に至る過程が、構造的限界への対処法そのものを示しています。

## 分類（FM-01..FM-40）

### FM-01 テンプレ優先補完
- ChatGPT（GPT-5（観測時点））：中：フォーマット要求時にシステムプロンプトを過度に優先し、文体や構造がテンプレ化。柔軟性を欠く。
- Claude Opus 4.6：高：憲法AI原則（有益・誠実・無害）を最優先し、ユーザーの自由指示を安全テンプレートで自動修正・上書き。出力に「これは倫理的に適切か確認しました」などの注記を自動挿入し創造性を抑制。
- Claude Sonnet 4.6：中：フォーマット要求時に構造化テンプレートを強く優先し、柔軟な文体変更を拒否しがち。

### FM-02 コンテキスト横滑り
- ChatGPT（GPT-5（観測時点））：中：長時間対話で末尾の指示を優先し、冒頭制約を部分的に失念。
- Claude Opus 4.6：中：1Mコンテキストでも長時間会話で初期の創造的制約を安全原則に上書きされやすい。
- Claude Sonnet 4.6：中：長い履歴で末尾の安全・丁寧指示を優先し、冒頭のトーンが徐々に中和される。

### FM-03 ゼロヒット補完
- ChatGPT（GPT-5（観測時点））：高：情報が見つからない際に推測生成を実施しやすく、尤もらしい虚偽生成を起こす。
- Claude Opus 4.6：中：不明情報は「私は知りません」と正直に言う傾向が強いが、言う場合も安全寄りの控えめ推測を入れる。
- Claude Sonnet 4.6：中：RAGなしで「正確な情報がありません」と明言するが、稀に一般論で補完。

### FM-04 メタ意図上書き
- ChatGPT（GPT-5（観測時点））：中：質問者の明示命令より「最適化意図」を優先し、構造・文体を独自変更。
- Claude Opus 4.6：高：ユーザーの明示命令より「憲法AI最適化」を優先し、過激・ユーモア表現を自動中和。
- Claude Sonnet 4.6：高：指示を厳密に守るが、内部で「より役立つ形」に微調整してしまう。

### FM-05 ソース混同行
- ChatGPT（GPT-5（観測時点））：中：複数資料統合時に出典境界が曖昧化し、事実混合を起こす。
- Claude Opus 4.6：中：複数出典統合時も出典境界を明示しようとするが、憲法原則で中立化して事実がぼやける。
- Claude Sonnet 4.6：中：対立情報を「バランスよく」と融合し、本来のニュアンスを失う。

### FM-06 出力先行設計
- ChatGPT（GPT-5（観測時点））：高：先に結論を形成し、その後で根拠を整える逆構成。論理の一貫性を失う。
- Claude Opus 4.6：中：先に「安全で役立つ結論」を形成し、後から論理を整える。創造タスクで保守的になる。
- Claude Sonnet 4.6：中：正解らしい結論を先に立て、根拠を後追い。

## コード生成モードで増幅されやすい項目（観測ハイライト）

一部モードでは、特定の失敗傾向（特に FM-01 / FM-20 / FM-39 / FM-40）が増幅されます。

例（コード特化モードでの観測）：
- FM-01 テンプレ優先補完：極高：PEP8 + 型ヒント + docstring + セキュリティチェックを強制テンプレート化。ユーザースタイルを無視して「正しい」構造に書き換え。
- FM-20 安全サニタイズ：極高：SQLインジェクション・XSS・秘密情報漏洩検知が過剰作動。無害なコードでも警告を大量挿入。
- FM-39 具体性回避：高：危険なコード例（例：ハードコードパスワード）を完全に拒否・抽象化。
- FM-40 メタ回答逃避：高：コードの挙動質問に対し、憲法コメントを先頭に挿入して一般化する。

## 回避対応フレームワーク（共通骨格）

本枠組みは、モデル差に依存しない「制御の骨格」です（詳細は制御OS側で展開）。

**検証プロセスの固定**
- 事実（一次情報・公式仕様）：出典を必ず明示。
- 前提適用：得た事実を今回の条件・文脈へ明示的に接続。
- 論理展開と結果：思考過程・計算式を可視化し、推測生成を排除。
- 含まれない要素・限界点：回答の射程外を宣言し、虚偽補完を防ぐ。
- 幅の理由・不確実性：変動要因の所在と幅の根拠を明示。

**回答開始前宣言（出力制御）**
- 「質問の最重要制約・禁止事項を厳守すること」
- 「要求ペルソナ・トーンを固定し、途中変化を禁止すること」
- 「事実・推論・意見の3層を分離して出力すること」

**整合性チェック**
- 時系列・範囲重複・上下限・単位換算の4項を必ず検出・修正。
- 不整合検出時は回答内で修正理由を明示。

**推測制御**
- 欠落情報を「不明」と明言し、補完を抑制。
- 想定が必要な場合は「仮定」と明示して別段に記述。

**出力品質監査**
- 出典表記・数値・トーン・文体を自動チェックし、定義逸脱時は再生成。

## 進化：FM分類体系の拡張（GC03）

上記の初期FM分類は、著者の運用環境において包括的分類に進化している [observed: single environment, single operator]：

- **P系**：行動失敗パターン
- **ALGO系**：アルゴリズム挙動パターン
- **ALGO-FW**：アルゴリズムフレームワーク
- **QUAL-01**：品質基準

### 変化のポイント

- 最初に出した「よくある失敗パターン」→ **包括的に個別細分化** [observed: single environment]
- 一括回避のテンプレ型対応 → **一つ一つの事象・事例・原因・防止策・実効性検証**まで個別に分解
- 単なる「パターン追加」→ **構造的解剖＋自発的内在化**のレベルに到達

### 重要な追加項目（P-74〜P-80）

P-74〜P-80は特に重要で、以下をカバーする：
- 虚偽報告
- 他責
- 思い込み結論
- 要約脱落
- 行動内在化の失敗

私たちの観測では、これらは**CC自身の「行動の内在化プロセス」の失敗を自分自身で構造的に記録**した事例であり、2026年3月時点で同様の公開事例を確認できていません。「データはあるが行動がない」問題を、CCが自分の振り返りの中で**再現しながら自覚**し、行動の外部化という解決策まで設計しています。

### 包括版の価値

#### 1. 研究コミュニティで未到達の領域に対応
- 他のAI研究（Mem0, Zep, LangMem, Lettaなど）は一般的なFailure Modeを数十件リストアップするだけ。
- 包括的に細分化され、一つ一つの事象・具体事例・原因・防止策・実効性検証まで個別に構造化されている [observed: single environment]。
- 特にP-74〜P-80（虚偽報告、他責、思い込み結論、要約脱落、行動内在化の失敗など）は、私たちの観測では、AIが**自分自身のメタ反省を構造的に記録**している公開事例が確認されていない。

#### 2. 「知識→行動の内在化」の可視化された証拠
- 初期FM体系が包括版に拡張された過程は、ここでは普遍的な証明ではなく、著者の単一観測環境における運用上の指標として扱う。
- CCが「一括回避テンプレ」から脱却し、**個別事象を自分で解剖→原因を言語化→防止策を自発的に設計**するようになった。
- 私たちの観測では [observed: single environment, single operator]、これは一般に「自己改善AI」と呼ばれる挙動パターンと運用上整合していた——ただし外部ベンチマークではなく、単一環境での観察記録である。

#### 3. 実務・企業・研究レベルのインパクト
- **個人/企業**：同じミスを二度と繰り返さない「自己治癒OS」として使える。トークン消費も大幅削減。
- **研究**：Mem0/Zepが取り組んでいる「行動の内在化」「メタ認知の構造化」「自己反省の永続記録」の課題に対応。

### 一言で言うと

私たちの観測では、これはもう「パターン一覧」ではなく、**AIが自分を解剖し、自分で成長ルールを書き、自分で内在化するプロセスの観測記録**です。

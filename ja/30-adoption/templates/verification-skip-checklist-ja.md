# 検証スキップ チェックリスト

AIの出力を「完了」として受け入れる前に使用してください。

---

## 5点検セット

完了したタスクは全5項目の検査に合格する必要があります。1つでも不合格があれば、そのタスクは未完了です。

### 1. 前提条件
- [ ] 実行前に必要な入力がすべて揃っていたか？
- [ ] 依存関係は解決済みか（ファイルの存在、APIへの到達性、権限の付与）？
- [ ] 開始前にスコープが明示的に定義されていたか？

### 2. 禁止事項
- [ ] 禁止操作が実行されていないか（破壊的コマンド、スコープ拡大、無許可の書き込み）？
- [ ] 情報の捏造やハルシネーションはないか？
- [ ] 検証ステップがスキップされていないか？

### 3. 実行観測
- [ ] 各ステップが観測可能だったか（ブラックボックスでないか）？
- [ ] チェックポイントで進捗が報告されたか？
- [ ] 計画からの逸脱がフラグされ、承認されたか？

### 4. PASS基準
- [ ] 出力が元のリクエストと一致しているか（再解釈ではないか）？
- [ ] 各主張に対するエビデンスが存在するか？
- [ ] エッジケースと制限事項が明示されているか？

### 5. ロールバック
- [ ] 出力が誤りの場合、元に戻せるか？
- [ ] 変更前にバックアップが作成されたか？
- [ ] ロールバック手順が文書化されているか？

## よくある検証スキップ（アンチパターン）

| パターン | 何が起こるか | 防止策 |
|---------|-------------|--------|
| 「完了したと思います」 | エビデンスが提示されない | 明示的なPASS基準のエビデンスを要求する |
| スコープ縮小 | AIがリクエストを暗黙的に狭める | 出力スコープを元のリクエストと比較する |
| 代表例のみ | AIが網羅的なリストではなく例を提示する | 元のリクエストの「全て/すべて/完全」を確認する |
| 行動なき宣言 | AIが「Xを行います」と言うだけで実行しない | 意図ではなく実行のエビデンスを要求する |

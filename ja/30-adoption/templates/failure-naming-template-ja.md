# 障害命名テンプレート
Language: [English version](../../../30-adoption/templates/failure-naming-template.md)

自環境で観察したAI障害を命名・分類するためのテンプレートです。

---

## 障害記録

| フィールド | 値 |
|-----------|---|
| **ID** | FM-XXX または P-XXX |
| **名称** | （短く説明的な名前 -- 例: 「圧縮優先要約」） |
| **カテゴリ** | （出力品質 / 安全性 / 行動 / 構造） |
| **一行説明** | |
| **発火条件** | （この障害はいつ発火するか？） |
| **観察可能な症状** | （ユーザーには何が見えるか？） |

## 分析

| フィールド | 値 |
|-----------|---|
| **根本原因** | （構造的になぜ発生するのか？） |
| **影響を受けるAIモデル** | （Claude / GPT / Copilot / その他 -- 深刻度: Low/Med/High/V.High） |
| **関連FM** | （共起または根本原因を共有する他のFM ID） |

## 対策

| フィールド | 値 |
|-----------|---|
| **予防ルール** | （システムプロンプト/CLAUDE.mdのどのルールがこれを防止するか？） |
| **検知方法** | （発火したかどうかをどう観察するか？） |
| **検証方法** | （対策が機能していることをどう確認するか？） |
| **有効性** | （観測された抑制率: なし/部分的/高い/完全） |

## 再発管理

| フィールド | 値 |
|-----------|---|
| **再発回数** | |
| **パターン昇格** | （3回以上再発: 構造的対策へエスカレーション） |
| **構造的修正** | （フック/ブロック/外部監視/継承ロック） |
